# Getting Started

## Installation

### Option 1: VS Code Extension (Recommended)

The fastest way to integrate STM32 Platform:

1. **Download** latest `.vsix` from [Releases](https://github.com/Gad9889/STM32-Platform/releases)
2. **Install** in VS Code: `Extensions → ... → Install from VSIX...`
3. **Open** your STM32 CMake project
4. **Run** command: `STM32 Platform: Integrate into Project`
5. **Select** peripherals → Done!

See [VS Code Extension](VS-Code-Extension) page for detailed instructions.

### Option 2: Manual Integration

```bash
# Clone repository
git clone https://github.com/Gad9889/STM32-Platform.git

# Copy files to your project
cp -r STM32-Platform/Inc/* YourProject/Inc/
cp -r STM32-Platform/Src/* YourProject/Src/

# Add to your build system
# CMakeLists.txt example:
include_directories(Inc)
add_executable(${PROJECT_NAME} 
    Src/main.c
    Src/stm32_platform.c
    Src/can.c
    Src/uart.c
    # ... other sources
)
```

## Prerequisites

- STM32 microcontroller (F0/F1/F4/F7/H7/G0/G4/L4)
- STM32CubeMX
- ARM GCC toolchain
- Your favorite IDE (VS Code, STM32CubeIDE, etc.)

## Your First Program

### Step 1: Configure with CubeMX

1. Open STM32CubeMX
2. Select your microcontroller
3. Enable peripherals you need:
   - **CAN**: Set to "Activated", enable interrupts
   - **UART**: Set to "Asynchronous", enable DMA (optional)
   - **ADC**: Enable channels, set to DMA if needed
4. Generate code

### Step 2: Write Code

Create `main.c`:

```c
#include "stm32_platform.h"

// External HAL handles (generated by CubeMX)
extern CAN_HandleTypeDef hcan1;
extern UART_HandleTypeDef huart2;

int main(void) {
    // HAL initialization (CubeMX generated)
    HAL_Init();
    SystemClock_Config();
    MX_CAN1_Init();
    MX_UART2_Init();
    
    // Initialize platform
    Platform.begin(&hcan1, &huart2, NULL, NULL, NULL);
    
    // Say hello!
    UART.println("STM32 Platform ready!");
    UART.printf("Version: %s\\n", Platform.version());
    
    // Main loop
    while (1) {
        // Process incoming CAN messages
        CAN.handleRxMessages();
        
        // Send periodic CAN message
        uint8_t data[] = {0x01, 0x02, 0x03};
        CAN.send(0x100, data, 3);
        
        HAL_Delay(1000);
    }
}
```

### Step 3: Build and Flash

Compile and flash to your STM32. Open a serial terminal at 115200 baud to see output!

## Common Issues

### Extension doesn't activate
- Make sure your workspace has a `CMakeLists.txt`
- Run: `STM32 Platform: Check Project Compatibility`

### Platform functions not found
- Verify you copied all files from `Inc/` and `Src/`
- Check include paths in your build system

### CAN not working
- Enable CAN interrupts in CubeMX
- Call `CAN.handleRxMessages()` in your main loop
- Check CAN transceiver wiring

### UART output not visible
- Verify UART baud rate (default 115200)
- Check TX pin connection
- Ensure `huart2` handle is passed to `Platform.begin()`
